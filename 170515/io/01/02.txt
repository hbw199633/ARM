fgetc()/fputc() 
优点：可以完成基本的文件读写。
缺点：单次操作io的数据小，效率低下。

文件流对象： 文本流   二进制流

字符串读写函数：fgets()/fputs() ==>针对文本流对象。

框架：
	fopen() ===>fgets()/fputs() ===>fclose();

读操作：fgets
char *fgets(char *s, int size, FILE *stream);
功能：该函数可以从指定的流对象中获取指定大小的数据到本地内存。
参数：s 用于存储数据的本地内存。
      size 要存储数据的本地内存大小，默认存储时候会自动 -1；
	  stream 获取数据的目标文件流指针
返回值：成功  获取到的数据地址
		失败/结尾  NULL

char *gets(char *s);  == fgets(s,size,stdin);

1、fgets的返回值是目标存储区的地址  == s  ?
2、size == sizeof(s) ==>实际大小是s内存大小还是 大小-1 怎么验证？
3、如果stream 流是stdin，会不会有内存溢出？
4、文件结尾判断  NULL, fgets也会读入文件中的换行符
5、fgets如果获取数据小于s大小且有'\n',则一次读取
	    如果获取数据大于s大小则按照sizeof(s)-1长度获取

注意：gets和scanf都存在内存溢出的问题。

写操作： fputs	
int fputs(const char *s, FILE *stream);
功能：该函数可以将s所在内存区中的数据，写入stream对应的文件流中。
参数：s 要写的目标字符串。
	  stream 要写的目标文件流对象
返回值：成功  返回>0的值
		失败 EOF

puts(s); == fputs(s+'\n',stdout);


1、s 能不能直接是字符串常量输出，或者字符串数组。
2、stream流对象如果是stdout的是有没有'\n'
3、如果用fputs输出和printf输出有什么差异？

二进制文件读写： fread/fwrite


size_t fread(void *ptr, size_t size, size_t nmemb, FILE *stream);
功能：该函数可以从指定的流对象中获取指定个数的单元数据到目标内存。
参数：ptr ==》void * ==》纯地址类型，表示存储数据的内存。
      size ==》int ===》代表要读取的数据单元大小。
	  nmemb ==>int ===> 代表要读取的数据个数。
	  stream 获取数据的源。如果是stdin表示从终端获取。
返回值：成功 返回 <= nmemb 的单元个数。
		失败/0  <= size   0 表示结尾。

 size_t   fwrite(const  void  *ptr,  size_t  size,  
                 size_t  nmemb,  FILE *stream);
 功能：该函数可以向指定的流对象文件中写入指定个数的二进制数据。
 参数：ptr要写入的目标数据
	   size 要写入的目标数据单元大小
	   nmemb 要写入的目标数据单元个数
	   stream 要写入的目标文件流指针
返回值：成功 <= nemeb的单元个数
		失败 < size 

二进制数据有两种：
1、自己构造二进制读写数据  ===》结构体
	读写程序都基于同一个结构体，读写都是基于该结构体。
	文件中存储的形式是二进制。

2、其他构造二进制读写数据  ===》其他二进制
	如果已知对方数据组织形式，则安装其格式读写。
	如果不知对方数据组织形式，按照指定单元读写。

格式化io

1、读操作  fscanf
int fscanf(FILE *stream, const char *format, ...);
功能：该函数可以从指定的流文件中获取指定格式的数据。
参数：stream 获取数据的目标文件流对象，如果是stdin从键盘终端获取。
	  format ... 与scanf类似的数据格式。
返回值：成功 获取的数据个数
		失败/文件结尾 EOF

2、写操作： fprintf
int fprintf(FILE *stream, const char *format, ...);
功能：该函数可以以指定格式的数据输出到目标流文件中。
参数：stream要输出数据的目标文件流对象，如果是stdout向屏幕终端输出
	  format ... 与pritnf类似的输出格式串
返回值：成功 返回输出的数据长度
		失败 EOF
